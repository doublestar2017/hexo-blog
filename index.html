Vite

发表于 2021-05-24 更新于 2021-05-25 分类于 vite

vite 的工作原理，它为什么这么快

index.html 文件

<body>
  <div id="app"></div>
  <script type="module" src="/src/main.js"></script>
</body>

<https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/script

.vue => 拦截请求 => 编译 => 返回给客户端

  1. vite 利用了客户端能力不用打包其他服务，原生的 ES Import 直接输出提高了冷启动速度。

  2. vite 按需编译当前页面需要的组件，而不需要打包整个 APP 的组件，这样的提升对比 cli 无疑是项目越大速度差距越大。

  3. HMR 更新速度不会和模块数量牵扯，vite 会让 HMR 一如既往的保持快速。

vite 对 TS 的支持

vite 内置 TS 的支持，开箱即用

<script lang="ts">
import { ref } from "vue";

export default {
  name: 'App',
  setup(){
    let hello = ref<string>("1");
    console.log(hello.value);
  }
}
</script>
